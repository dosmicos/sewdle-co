❓ FAQ RÁPIDO
═════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════

P: ¿A qué hora se ejecuta un cron?
R: En el horario UTC especificado. Para Colombia, resta 5 horas.

P: ¿Puedo cambiar el horario después?
R: Sí, crea nueva migración con cron.unschedule() y cron.schedule() nuevo.

P: ¿Cuál es el intervalo mínimo?
R: 1 minuto. Para menos, usa Cloud Functions.

P: ¿Dónde veo si se ejecutó?
R: Supabase Dashboard → Functions → Logs o `npx supabase functions logs`

P: ¿Qué pasa si falla?
R: El cron lo reintenta. Revisa logs para ver el error específico.

P: ¿Puedo desagendar un cron?
R: Sí, en SQL Editor: SELECT cron.unschedule('nombre-del-job');

P: ¿Funciona de noche?
R: Sí, crons se ejecutan 24/7 automáticamente.

📊 MONITOREO BÁSICO (Hacer cada semana)
═════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════════

Cada VIERNES por la mañana:
  1. Abrir Supabase Dashboard → Functions → Logs
  2. Buscar tus funciones automatizadas de la semana
  3. Verificar:
     ✓ Status 200 (éxito)
     ✓ Sin errores
     ✓ Timestamp coincide con horario agendado
  4. Si ves ❌: Investigar causa en TROUBLESHOOTING

Cada PRIMER VIERNES DEL MES:
  1. Verificar que la función se ejecutó las veces esperadas
     Esperadas = días en mes / intervalo
     Ej: 31 días / 1 = 31 ejecuciones diarias
  2. Verificar que los datos se actualizaron correctamente
  3. Comparar resultados con mes anterior
